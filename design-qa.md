# MUCHI 모바일 전용 레이아웃 QA

검증일: 2026-07-15
기준 URL: `http://localhost:3001`
제품 기준: 최대 430px 모바일 앱 캔버스, 데스크톱 전용 레이아웃 없음

## 구현 계약

- 실제 모바일에서는 앱이 `100vw × 100dvh`를 사용한다.
- 480px 이상 브라우저에서는 430px 앱 캔버스를 가운데 배치하고 상하좌우 16px 여백을 둔다.
- 앱 프레임은 `44px 헤더 / 내부 스크롤 콘텐츠 / 70px 하단 내비게이션` 구조다.
- 소형 화면 보정은 앱 컨테이너 기준 419px 이하에서만 적용한다.
- 브라우저 폭과 무관하게 챕터는 2열, 기억은 1열, 챕터 상세는 압축형 레이아웃을 유지한다.
- 다이얼로그, 태그 시트, 미니·전체 플레이어, 토스트는 앱 프레임 내부에 제한한다.

## 자동 회귀 검증

- 활성 스타일에서 `vw` 단위를 제거하고 `cqw` 및 고정 모바일 범위로 교체했다.
- 비활성 `@media (max-width: 0px)` 레거시 테마를 삭제했다.
- 430px 캔버스, 16px 외부 여백, 28px 프레임 반경과 내부 스크롤 구조를 테스트로 고정했다.
- 넓은 화면에서 프레임 전체가 움직이지 않도록 View Transition 대상을 `route-content`로 제한했다.
- URL별 내부 스크롤 위치 저장, 전진 이동 상단 초기화, 뒤로 가기 복원 계약을 소스 회귀 테스트로 고정했다.

## 브라우저 런타임 검증

1280×720 브라우저에서 앱 프레임은 `430×688px`, 위치는 `x=425, y=16`으로 측정됐다. 헤더 44px, 콘텐츠 572px, 하단 내비게이션 70px이 정확히 유지됐다.

오버레이 측정 결과:

- 링크 추가 다이얼로그 백드롭: `428×686px`, 프레임 테두리 안쪽에 정확히 고정
- 태그 선택 시트: 폭 428px, 프레임 하단에 고정
- 미니 플레이어: 폭 396px, 프레임 내부 배치 및 콘텐츠 하단 여백 68px 확보
- 전체 플레이어: 백드롭 `428×686px`, 내부 스크롤 잠금과 첫 조작 요소 포커스 확인
- 브라우저 오류 로그: 오류·경고 없음

## 플레이리스트 회귀

- URL에 Spotify가 미리 지정된 상태에서도 `곡 확인 → 서비스 선택 → 매칭 확인` 3단계를 순서대로 통과했다.
- 매칭 결과가 실제 연동 전 미리보기임을 명시했다.
- 완료 상태를 “플레이리스트 내보내기 준비 완료”로 표현했다.
- 외부 음악 서비스 호출이나 로컬 아카이브 데이터 변경 없이 완료 화면까지 도달했다.

## 하위 챕터 회귀

- 챕터 상세에서 현재 위치 경로, 하위 챕터 수, 빈 상태, 생성 진입점을 확인했다.
- 생성 시트는 이름·설명·분위기 색상을 제공하고, 저장 전에는 로컬 아카이브를 변경하지 않는다.
- 430px 앱 프레임에서 하위 챕터 영역의 가로 넘침은 `0px`이었다.
- 생성 시트의 취소·완료 액션은 하단 내비게이션 위에 고정되어 첫 화면부터 노출된다.
- 부모 삭제 시 직계 자식은 한 단계 승격되고, 자식의 곡·태그·기억은 유지되는 회귀 테스트를 통과했다.
- v1–v3의 평면 챕터 데이터는 v4의 루트 챕터로 자동 이전된다.

## 최종 UX 리팩터링 리뷰

- HOME·CHAPTERS·ADD·SEARCH는 루트 탭으로 취급해 불필요한 상단 뒤로가기를 제거했다. 챕터 상세·기억·설정처럼 실제 하위 화면에서만 뒤로가기를 제공한다.
- 기억 편집에서 곡별 `성격 문장`과 `기억한 시기(월·계절, 선택 연도)`를 직접 기록할 수 있다. 태그·메모와 함께 같은 곡의 챕터별 맥락으로 독립 저장된다.
- 챕터 상세은 `하위 챕터 → 수록곡 → 플레이리스트 내보내기` 순서로 재배치해 개인 아카이브를 외부 서비스보다 먼저 탐색하게 했다.
- 플레이리스트의 뒤로가기를 상단 버튼 하나로 통합했다. 2·3단계에서는 이전 단계로 돌아가고 1단계에서만 플레이리스트를 나가며, 단계가 바뀔 때 콘텐츠 스크롤도 상단으로 복원된다.
- 전체 플레이어의 `닫기`는 미니 플레이어로 축소하고, 상단의 `플레이어 종료`는 오디오와 미니 플레이어를 함께 완전히 닫는다. 짧은 프레임에서도 종료 액션은 스크롤 없이 노출된다.
- 플레이어를 완전히 종료하면 제거되는 미니 플레이어 대신 본문으로 키보드 포커스를 복원한다.
- 빈 자동 월별 챕터는 목록에서 숨기고, 곡이 있는 월별 챕터도 수동 저장 목적지에서는 제외해 등록일 기반 분류가 오염되지 않게 했다.
- 월별 챕터는 저장 구조를 바꾸지 않고 화면에서 `연도 + 월`로 표시해 다른 해의 같은 달을 구분한다.
- HOME 캐러셀에 44px 이전·다음 버튼과 현재 곡 알림을 추가해 스와이프를 쓸 수 없는 스크린리더·스위치 사용자도 모든 기억을 탐색할 수 있다.
- 사용되지 않던 `TrackLine` 미리듣기 분기, 항상 참이던 수화 props, 무시되던 빈 상태 아이콘, 사용되지 않는 색상 토큰과 앨범 폴백 스타일을 제거했다.
