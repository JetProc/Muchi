# 뮤키 (Muchi)

> **챕터로 쌓아 올린 나만의 음악 세계**

뮤키는 좋아했던 음악을 제목이 아니라 **기억, 감정, 장면**으로 남기고 다시 찾는 모바일 음악 아카이브입니다.

`새벽 산책`, `첫 자취방`, `비 오는 날에 듣던 재즈`처럼 나만 아는 단서로 곡을 기록합니다. 같은 곡이라도 어떤 시기와 장면에서 들었는지에 따라 서로 다른 챕터에 담아, 음악과 함께 내 시간을 쌓아 갑니다.

## 왜 뮤키인가요?

음악 앱의 좋아요와 플레이리스트는 곡을 모으기에는 좋지만, **왜 그 곡이 좋았는지**와 **어느 순간의 음악이었는지**는 쉽게 사라집니다.

- 곡명이나 아티스트는 잊었지만, 들었던 장면은 기억날 때
- 좋아요 목록이 쌓여도 다시 꺼내 듣기 어려울 때
- 장르나 분위기보다 나만의 기준으로 음악을 정리하고 싶을 때
- 다른 사람의 취향보다, 그 사람이 음악을 좋아한 맥락을 발견하고 싶을 때

뮤키는 음악을 더 많이 저장하는 서비스가 아니라, 좋아했던 음악을 더 오래 내 것으로 남기는 서비스입니다.

## 뮤키의 방식

### 1. 발견한 순간, 부담 없이 포착합니다

YouTube Music 또는 Apple Music에서 공유한 링크를 가져오거나 서비스 안에서 곡을 찾습니다. 기록을 시작하는 데 긴 설명은 필요하지 않습니다.

### 2. 나만의 언어로 맥락을 더합니다

태그와 메모는 선택 사항입니다. `운동할 때`, `기분 전환`, `여름밤`처럼 다시 찾을 수 있는 짧은 단서를 남기고, 필요하다면 애정도를 색 점으로 표시합니다.

### 3. 챕터로 음악 세계를 만듭니다

곡을 시기·장면·감정·취향에 따라 챕터에 담습니다. 같은 곡도 챕터마다 다른 기억을 가질 수 있어, 하나의 곡이 여러 삶의 장면으로 남습니다.

### 4. 시간이 지난 뒤, 기억으로 다시 찾습니다

곡명뿐 아니라 태그·메모·챕터를 통합 검색하고, 남겨 둔 단서를 따라 예전의 음악을 다시 만납니다.

## 한 곡이 기록이 되는 과정

1. 음악 앱에서 좋아하는 곡을 발견합니다.
2. 공유 링크를 뮤키로 가져오거나, 뮤키에서 곡을 검색합니다.
3. 곡을 담을 챕터를 고르거나 새로 만듭니다.
4. 필요하면 태그, 메모, 애정도를 더합니다.
5. 나중에는 기억나는 단어로 그 곡과 당시의 장면을 다시 찾습니다.

## 핵심 기능

| 경험 | 설명 |
| --- | --- |
| 곡 가져오기 | 서비스 내 검색과 YouTube Music·Apple Music 공유 링크를 통해 같은 기록 흐름으로 곡을 추가합니다. |
| 개인 태그와 메모 | 자유로운 태그와 짧은 메모로 곡을 다시 찾을 나만의 단서를 남깁니다. |
| 애정도 | `최애`·`자주 찾음`·`좋아함`을 빨강·주황·노랑 점으로 표시하고, 챕터 안에서 애정도 순으로 정렬할 수 있습니다. |
| 챕터 | 곡을 장면과 시간의 단위로 엮습니다. 하위 챕터, 대표 이미지, 설명, 공개 범위를 설정할 수 있습니다. |
| 보관함 | 아직 정리하지 않은 곡을 모아두고, 여러 곡을 한 챕터에 한 번에 넣을 수 있습니다. |
| 다시 찾기 | 곡·아티스트·챕터·태그·메모를 통합 검색하고, 태그별로 음악을 모아봅니다. |
| 회고 | 이맘때, 타임라인, 무작위 방식으로 예전에 기록한 음악을 다시 꺼냅니다. |
| 탐색 | 다른 뮤커가 공개한 챕터를 둘러보고, 곡 목록과 공개된 기록의 맥락을 함께 발견합니다. |

## 공개는 선택입니다

뮤키의 중심은 개인 아카이브입니다. 챕터는 공개 또는 비공개로 정할 수 있고, 공개 챕터 안에서도 각 곡 기록의 태그와 메모는 별도로 공개 여부를 정할 수 있습니다. 비공개 기록의 개인적인 맥락은 탐색 화면에 노출되지 않습니다.

## 현재 지원 범위

- 곡 검색과 YouTube Music·Apple Music 공유 링크 가져오기를 지원합니다.
- Spotify 링크 가져오기는 현재 준비 중입니다. 지금은 서비스 내 검색으로 같은 곡을 찾아 기록할 수 있습니다.
- Apple Music·Spotify·YouTube Music으로의 플레이리스트 내보내기는 프로토타입 단계에서 준비 중이며, 실제 계정에 플레이리스트를 생성하지 않습니다.
- 뮤키는 스트리밍 서비스가 아닙니다. 곡의 원본 재생은 연결된 음악 서비스에서 이어집니다.

## 로컬에서 보기

Node.js `22.13.0` 이상이 필요합니다. Supabase 프로젝트의 URL과 publishable key를 `.env.example`을 참고해 `.env.local`에 설정한 뒤 실행합니다.

```bash
npm install
npm run dev
```

```bash
npm run lint
npm run typecheck
npm test
```

## Supabase 마이그레이션

새 Supabase 환경에는 `supabase/migrations/`의 SQL을 시간순으로 모두 적용합니다. 기존 운영 프로젝트를 CLI와 연결할 때는 과거 이력을 다시 실행하지 말고, [마이그레이션 정합성 안내](docs/SUPABASE_MIGRATION_RECONCILIATION.md)를 따라 적용 이력만 먼저 맞춥니다.

곡별 기록 사진과 챕터 공유 기능을 배포하려면 다음 항목도 필요합니다.

- `supabase/migrations/20260723103000_private_record_photos.sql` 적용
- 로컬과 Vercel Production에 서버 전용 `SUPABASE_SERVICE_ROLE_KEY` 설정
- 서비스 역할 키에는 `NEXT_PUBLIC_` 접두사를 붙이지 않고 클라이언트 코드에서 사용하지 않기
