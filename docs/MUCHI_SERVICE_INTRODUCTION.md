# 뮤키(Muchi) 서비스 소개

> **챕터로 쌓아 올린 나만의 음악 세계**

## 한 줄 소개

- 음악을 **제목이 아닌 기억·감정·장면**으로 기록하고 다시 찾는 개인 음악 아카이브

## 이런 사람을 위한 서비스

- 좋아요한 곡이 많아져서 나중에 다시 찾기 어려운 사람
- “비 오는 날 들은 노래”처럼 음악을 상황과 감정으로 기억하는 사람
- 플레이리스트를 넘어, 자신만의 기준으로 음악을 정리하고 싶은 사람
- 다른 사람의 취향과 기록을 따라 새로운 음악을 발견하고 싶은 사람

## 해결하는 문제

- 스트리밍 서비스의 저장 목록은 곡이 많아질수록 맥락이 사라짐
- 곡명·아티스트를 잊으면 원하는 음악을 다시 찾기 어려움
- 플레이리스트는 곡을 담지만, 왜 좋아했는지와 어떤 순간의 음악인지는 담기 어려움

## 핵심 기능

### 1. 링크로 곡 가져오기

- YouTube Music·Apple Music 공유 링크를 붙여 넣어 빠르게 곡 추가
- 스트리밍 앱에서 발견한 순간, 바로 기록 화면으로 이동
- 서비스 안에서 직접 검색해 추가하는 흐름과 동일한 기록 경험 제공

### 2. 태그와 메모로 기억 기록

- `새벽 산책`, `비 오는 날`, `우연히 들은 노래`처럼 개인적인 태그 추가
- 태그·메모는 모두 선택 사항으로, 가볍게 저장하거나 자세히 기록 가능
- 제목이 기억나지 않아도 내가 남긴 기억을 단서로 다시 찾기

### 3. 애정도 표시

- 곡마다 좋아하는 정도를 색상 점으로 간단히 표시
- 빨강 `최애` · 주황 `자주 찾음` · 노랑 `좋아함`
- 챕터 안에서 애정도 높은 순으로 곡 정렬 가능

### 4. 챕터로 음악 세계 만들기

- 시기·장면·감정·취향 등 나만의 기준으로 챕터 생성
- 예시: `2026년 여름의 밤`, `혼자 걷고 싶은 날`, `우연히 발견한 음악`
- 챕터별 대표 이미지, 설명, 공개·비공개 설정 지원

### 5. 보관함에서 나중에 정리

- 지금 바로 기록하기 어려운 곡을 먼저 보관
- 여러 곡을 선택해 한 챕터에 한 번에 넣기
- 기록·삭제 등 필요한 작업을 선택 후 일괄 처리

### 6. 탐색으로 취향 발견

- 공개된 다른 뮤커의 챕터 탐색
- 곡 목록뿐 아니라 챕터의 주제, 태그, 메모, 애정도까지 함께 확인
- 차트가 아닌 사람의 맥락을 따라 음악 발견

## 서비스가 만드는 경험

1. 음악 앱에서 마음에 드는 곡을 발견한다.
2. 공유 링크로 뮤키에 가져온다.
3. 챕터를 고르고, 필요하면 태그·메모·애정도를 남긴다.
4. 시간이 지난 뒤, 곡명이 아닌 기억으로 다시 찾는다.
5. 기록이 쌓여 나만의 음악 세계가 된다.

## 뮤키의 방향

- 더 많은 곡을 저장하게 하기보다, 좋아했던 음악을 더 오래 기억하게 하기
- 음악을 단순한 목록이 아니라 삶의 장면과 연결된 기록으로 남기기
- 취향을 공개할 때도 곡만이 아니라 그 곡을 좋아한 이유와 맥락을 나누기
